PHASE 2 — BACKEND FEATURES
================================================================================

CARD PACK / COLLECTION SYSTEM
------------------------------
- POST /api/packs/open
    Open a pack, generates cards with weighted rarity rolls,
    applies stat multipliers, credits to user_collection

- GET /api/collection
    Get authenticated user's full card collection

- GET /api/collection/:userId
    View another user's public collection

CARD LEVELING
-------------
- Cards gain XP from battles
- Level 1-10; each level boosts all stats by ~8%
- Combat Power = weighted average of stats x rarity multiplier x level bonus

TRADING SYSTEM
--------------
- POST   /api/trade/list              List a card for trade (coin price or card-for-card)
- GET    /api/trade/listings          Browse all active listings (filter by rarity, fighter, price)
- POST   /api/trade/offer             Make an offer on a listing
- PUT    /api/trade/accept/:listingId Accept offer, swap ownership atomically
- DELETE /api/trade/cancel/:listingId Cancel own listing

BATTLE SYSTEM
-------------
Battle Resolution Algorithm:
  1. Each fighter rolls base stats + level bonus + rarity multiplier
  2. 3 rounds: each round picks a random stat category (power, speed, durability, IQ)
  3. Add +-10% randomness (RNG element)
  4. Win 2+ rounds = match winner
  5. Winner gains XP and coins; loser gains smaller XP consolation

Endpoints:
- POST /api/battle/challenge     Challenge another user or AI (select your card, pick opponent)
- GET  /api/battle/history       Battle log for current user
- GET  /api/battle/leaderboard   Ranked by wins / combat power

STORE
-----
- GET  /api/store/packs   List available card packs
- POST /api/store/buy     Purchase a pack with coins or gems
- GET  /api/wallet        Get user coin and gem balance

================================================================================
NOTE ON DATABASE DESIGN
================================================================================

The existing ufc_fighter and ufc_weight_fighter tables stay as the source of
truth for raw fighter data. The fighter_card table references them but adds
rarity, level, and card-specific fields.

This means the same fighter (e.g. Jon Jones) can exist as a Common card AND
a Legendary card simultaneously — each with different stats, art, and abilities.
